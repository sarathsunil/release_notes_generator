2                                                 Introduction

This document describes the contents of a major product release of the Identity and Access Management, for T-Mobile, USA. It contains information about known dependencies and limitations that must be observed during installation or use of the release. It also describes corrections made due to trouble reports and change requests.

2.1                                          Naming Convention for Release Packaging

                   IAM_17-Rel_1 (5.1.0)

                   IAM_X-Rel-n (Y.M.m) **n = release counter

IAM_PROD-Rel_n (release, drop, hotfix) **n = release counter

 

IAM           Project Name, Identity and Access Management

X                ECE product version the solution, ex; (15, 17, 19…)

n                 Release counter

Y                Release within the product version (Major Functionality)

M               Drop within the release (Major Functionality)

m               Minor change, typically used for hotfixes and corrections

Example; IAM_17-Rel_1 (5.1.0)

In this example this is the first release to the customer, which includes ECE product version 17, first drop of the fifth release with no Emergency corrections or Hotfixes

2.2                                          Purpose

The purpose of this document is to communicate major and minor product releases (and Changes) to the Identity and Access Management – IAM 17 release 7 drop 2 hotfixes merged from Release 7 drop 2 (IAM_17-Rel_19(7.2.1) release)
